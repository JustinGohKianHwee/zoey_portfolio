# CLAUDE.md

## Project

This is a premium architecture portfolio website for **Sze Zo Ey**, a Master’s Architecture student.

The goal is to create a refined, employability-focused portfolio that presents her architectural work clearly and beautifully to architecture firms, studios, recruiters, academic reviewers, and collaborators.

Quality bar: this should feel like a **$10k custom architecture portfolio website** — polished, editorial, spatial, premium, and thoughtfully designed.

It should not feel like a generic student portfolio, Adobe Portfolio template, Wix/Squarespace site, or AI-generated landing page.

## Design Freedom

Prioritise strong design judgment.

Use the `/ui-ux-pro-max` skill for visual design, UX, layout, typography, spacing, motion, information architecture, and portfolio storytelling decisions.

Do not overfit to these instructions. Use them as direction, not rigid rules.

Before coding, propose the best portfolio strategy and design direction based on Zoey’s work, architecture portfolio conventions, and employability goals.

## Portfolio Intent

This website should communicate:

* who Zoey is
* what kind of architectural designer she is becoming
* her strongest projects
* her design thinking and process
* her technical and visual skills
* why she would be valuable to an architecture firm or design studio

This should be more than a gallery. It should present her work as curated architectural case studies with clear hierarchy, strong visuals, and concise storytelling.

Do not invent achievements, awards, internships, or project outcomes. If information is missing, mark it as `[needs Zoey input]`.

## Project Showcase Principle

Design the website and its components to showcase each architecture project in the strongest possible way.

Do not force every project into the same rigid layout if the work would be better presented differently. Use flexible, reusable components that can adapt to different project types, image sets, and storytelling needs.

For each project, decide the best presentation strategy based on the available content:

* hero image or hero spread
* project overview
* concept statement
* plans / sections / elevations
* diagrams
* renders
* process images
* material / environmental / structural strategy
* gallery moments
* captions and annotations
* final outcome

Prioritise architectural clarity. The viewer should quickly understand the project’s concept, spatial logic, site response, programme, process, and final design outcome.

The goal is not just to make the website look beautiful. The goal is to make each project feel more impressive, more legible, and more employable.


## Technical Direction

Preferred stack:

* Next.js
* TypeScript
* Tailwind CSS
* Vercel deployment

Keep the codebase clean, maintainable, and easy to update.

Structure project content in a way that makes it easy to add, edit, or reorder projects later.

Do not rely permanently on Adobe CDN image links. Use placeholders only when necessary and make it easy to replace them with Zoey’s original high-resolution images later.

## Workflow

Before implementation:

1. Use `/ui-ux-pro-max`.
2. Ask clarifying questions where needed.
3. Audit the existing Adobe Portfolio content.
4. Propose a portfolio strategy.
5. Propose 2–3 strong design directions.
6. Recommend the best direction.
7. Wait for approval before major coding.

During implementation:

* explain major design and code decisions
* preserve useful content
* avoid generic templates
* prioritise polish, clarity, responsiveness, accessibility, and performance

After implementation:

* summarise what changed
* explain the site structure
* explain how to add or edit projects
* explain how to deploy on Vercel
* list what Zoey should review before publishing

## Overall Principle

Give Claude and the design skill enough freedom to create something excellent.

The final site should feel like a curated architectural portfolio: elegant, spatial, editorial, employability-focused, and distinctly better than the current Adobe Portfolio.
